# Auto-Alt-Enter (v1.0.1)

## 概要
「Antigravity」のウィンドウを自動検知し、定期的に `ALT+ENTER` を送信する自動化プログラムです。
また、指定したGitリポジトリの履歴を監視し、新しいコミットが検知された場合に、自動的に送信処理を停止する機能を持っています。

## ファイル構成
*   `auto_alt_enter.py`: コアの処理を行うPythonスクリプト。
*   `start.bat`: 監視間隔やターゲットリポジトリのパスなどを設定して、簡単に起動するためのバッチファイル。

## 使い方
1.  **設定の変更**: `start.bat` をテキストエディタで開き、以下の環境変数を設定します。
    *   `INTERVAL`: `ALT+ENTER` を送信する間隔（秒）。デフォルトは3秒です。
    *   `REPO`: 監視対象のGitリポジトリの絶対パス（例: `C:\path\to\repo`）、またはカレントディレクトリ（`.`）。空のままにすると、コミット監視による自動停止機能は無効になります。
2.  **実行**: `start.bat` をダブルクリックして実行します。
3.  **依存関係の自動解決**: 初回実行時、必要なPythonパッケージ（`keyboard`, `pywin32`）がインストールされていない場合は自動的に `pip` 経由でインストールされます。
4.  **停止方法**:
    *   Gitリポジトリを監視している場合: 該当リポジトリで**新しいコミット**が行われると自動的に停止します。
    *   監視していない場合、または手動で止める場合: 起動しているコマンドプロンプト上で `Ctrl+C` を押してください。

## ⚠️ 注意点
*   **フォアグラウンドの強制奪取**: このツールは対象のウィンドウをフォアグラウンド（最前面）に引き上げてから物理的なキー入力（`ALT+ENTER`）をエミュレートします。手動で他の作業をしている最中にツールが動作すると、作業中のウィンドウからフォーカスが奪われ、意図せぬ誤動作を引き起こす可能性があります。
*   **キーロックのリスク**: スクリプト内でALTやENTERのリリース処理を入れていますが、ユーザーが同時にキーボードを操作している場合や、他のマクロツールが競合した場合、キーが押しっぱなしになったままになるリスクがあります。
*   **Windows専用**: 本スクリプトは `win32gui` や `ctypes.windll` などのWindows独自のAPIを使用しているため、他のOS（Mac/Linux）では動作しません。
*   **自動停止なしの無限ループ**: `REPO` を空に設定した場合、手動で停止させない限り永続的に `ALT+ENTER` を送信し続けます。起動させたまま放置しないよう注意してください。
